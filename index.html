Phase 1: Establishing Requirements

As per the Interaction Design Process (Figure 5.1), this phase identifies "What is Wanted" through user observation and technical constraints.

User Scenarios

The Correction: A user draws a boundary but realizes a vertex is misplaced. The 'm' (Move) requirement allows high-precision adjustment.

The Refinement: When an accidental point is created, the 'd' (Delete) requirement enables targeted error recovery.

Ethnographic Insight

Observing manual geography plotting shows that mistakes are common. To reduce "Cognitive Load," the requirements prioritize Editability over a "one-and-done" drawing approach.

Functional Requirements Specification

Primary Actions

  • B Begin a new line sequence
  • M Move the nearest vertex
  • D Delete the nearest vertex
  • R Refresh / Redraw Canvas
  • Q Quit Application

System Logic

  • Euclidean distance for point selection
  • Buffer for up to 100 active polylines
  • Coordinate tracking via 2D array structure
