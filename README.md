## Project Overview
This repository represents my contribution to the polyline editor project. My primary focus is on **Phase 1: Requirements Analysis** of the Interaction Design Process as defined by **Figure 5.1 (Bird’s-eye view)**.

According to the design flowchart, my role is the foundation of the project: establishing "What is Wanted". By analyzing user needs through ethnography and observation, I translated manual coordinate plotting tasks into functional system requirements that allow for non-destructive editing.

## My Contributions
My specific phase documents include:
* `requirements.md`: An analysis of the problem statement, field observations, and the functional specification for keyboard triggers.
* `scenarios.md`: Detailed conceptual models and user flows for correcting drawing errors and data refinement.
* `index.html`: A functional webpage documenting my requirements findings and hosting the shared application code.

## Challenges and Confusions
* **Requirement Mapping:** Translating manual coordinate plotting tasks (influenced by geography curricula) into specific digital keyboard interrupts.
* **Scope Management:** Defining a clear boundary for the "closest point" detection using Euclidean distance to ensure the user interaction feels natural.
